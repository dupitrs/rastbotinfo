Vienā teikumā: tu vadi visu vienā Google tabulā; GitHub serveros fonā strādā 3 nelieli roboti, kas paši sūta personalizētus e-pastus, apstrādā atbildes un uztur sistēmu dzīvu — bez tava datora, bez tavas klātbūtnes.

Kā viss savienojas

Augšējā josla = e-pastu sūtīšana. Apakšējā = atbilžu apstrāde. Cipari atbilst skaidrojumam zem diagrammas.

1 Tu pievieno kontaktus un ieslēdz sūtīšanu tabulā.
2 Sūtītājs nolasa iestatījumus + nākamo nesūtīto kontaktu.
3 Nosūta personalizētu e-pastu caur Gmail (SMTP); ja nedēļu nav atbildes — vēl vienu īsu follow-up tajā pašā sarunā.
4 E-pasts nonāk pie kontakta (saņēmēja).
5 Kontakts atbild — atbilde aiziet atpakaļ uz iesūtni.
6 Atbilde nonāk Gmail iesūtnē.
7 Atbilžu bots izlasa iesūtni (IMAP).
8 Ja atbilde nav "nē" → pārsūta uz rast.studija@gmail.com.
9Atzīmē tabulā atbildeja; ja atbilde bija „nē" — arī situacija = „atbildēja: nē".

3 roboti — kas ko dara

Visi trīs ir mazas Python programmas, kas dzīvo vienā GitHub repo un palaižas paši uz GitHub serveriem pēc grafika.

⏰

1. Sūtītājs

sender_job.py

Galvenais robots. Katrā palaišanā paņem nākamo nesūtīto kontaktu, izvēlas pareizo vēstuli pēc profesijas, personalizē ar vārdu, nosūta un atzīmē „nosūtīts". Ja pēc nedēļas atbildes nav, nosūta īsu atgādinājumu (follow-up) tajā pašā sarunā — tikai vienu reizi. Sūta pa vienam ar pauzēm; dienas skaits pakāpeniski aug (10, 12, 14…), lai sargātu reputāciju. Ja GitHub kādu palaišanu izlaiž un diena atpaliek, nākamā palaišana nosūta 2–3 uzreiz (ar pauzi pa vidu) — diena tāpat sasniedz savu skaitu.

Kad: katru dienu, 8:00–19:00, izkliedēti pa visu logu Kur: GitHub Actions Aiztiek: tabula + Gmail (sūta)
✉️

2. Atbilžu bots

replies_job.py

Lasa Gmail iesūtni un reaģē uz kontaktu atbildēm. Jebkura atbilde → statuss atbildeja (tam cilvēkam vairs neko nesūtīs, arī follow-up ne). Ja atbilde ir „nē" → nepārsūta un kolonnā situacija pieraksta „atbildēja: nē"; citādi pārsūta vēstuli uz rast.studija@gmail.com. Aiztiek tikai zināmus kontaktus; neapstrādā divreiz; izlaiž atvaļinājuma auto-atbildes.

Kad: reizi stundā, 24/7 Kur: GitHub Actions Aiztiek: Gmail (lasa) + tabula + pārsūta
🔄

3. Keepalive

keepalive.yml

Mazs palīgs. GitHub automātiski atslēdz grafikus pēc 60 dienām bez aktivitātes. Šis reizi mēnesī izdara sīku izmaiņu repo, lai sistēma neapstātos pati no sevis. Tu par to nedomā — tas dara pats.

Kad: reizi mēnesī Kur: GitHub Actions Aiztiek: tikai repo (sīks commit)

Kas ir „GitHub Actions" (vienkārši)

☁️ Bezmaksas mākoņa dators, kas pieder GitHub

Roboti nestrādā uz tava datora — tavam datoram nav jābūt ieslēgtam. Tā vietā GitHub pēc grafika uz dažām sekundēm ieslēdz īslaicīgu datoru savos serveros, lejupielādē mūsu kodu, palaiž to un izslēdz. Tas notiek pats, fonā, visu diennakti.

1 · Pienāk grafika laiks 2 · GitHub ieslēdz īslaicīgu datoru 3 · Lejupielādē kodu no repo 4 · Palaiž robotu (~30 sek) 5 · Izslēdz datoru

Grafiku nosaka „cron" izteiksme (piem. „katru stundu"). Robots katrā palaišanā pats izlemj, vai šobrīd kaut ko darīt — pārbaudot iestatījumus tabulā.

Lietotāja ceļvedis

Visu vadi Google tabulā (cilne iestatijumi) — nekas nav „jāprogrammē". Izmaiņas stājas spēkā nākamajā robota palaišanā.

▶️ Ieslēgt / apturēt sūtīšanu

aktivs = jā lai sūta, nē lai apturētu.

⚙️ Mainīt dienas tempu

Kāpinājums aug pats (rampa=jā). Lai fiksētu skaitu: rampa=nē + dienas_limits.

🧱 Uzlikt maksimumu

rampa_maks = piem. 30 — augs līdz 30/dienā un paliks.

➕ Pievienot kontaktus

Jaunas rindas cilnē kontakti. Robots tos pamanīs.

🚫 Izslēgt kādu manuāli

Tā rindā statuss = atbildeja.

📝 Piezīmes par atbildētājiem

Kolonna situacija cilnē kontakti — tava brīvā teksta kolonna (ko sarunājāt, kas tālāk). Bots tur raksta tikai „atbildēja: nē".

🌙 Sūtīt arī brīvdienās

tikai_darba_dienas = nē (vai jā tikai darba dienās).

📬 Atbilžu apstrāde

atbildu_apstrade = jā; parsutit_uz = uz kuru e-pastu pārsūtīt.

🔁 Atkārtotā ziņa (follow-up)

followup = jā/nē; followup_dienas = pēc cik dienām bez atbildes (tagad 7).

👀 Sekot līdzi

Cilne log — kas nosūtīts un kad. apstradatas_atbildes — apstrādātās atbildes.

🛟 Ja kaut kas nestrādā

Pārbaudi aktivs=jā un laiku 8–19. GitHub → cilne Actions rāda, vai palaišanas ir zaļas.
